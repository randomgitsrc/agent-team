## 待解决

### Superpowers + Claude Code + omnirouter/kimi thinking 兼容性

**问题**：Claude Code thinking 模式要求 reasoning_content 字段，但 kimi-k2.5 不返回该字段，导致 API 400 错误。

**已尝试方案**：
- alwaysThinkingEnabled: false ❌
- CLAUDE.md 注入 ❌
- 环境变量 ANTHROPIC_THINKING_ENABLED=false ❌

**根因**：thinking 在 Claude Code 内部硬编码，无法通过配置关闭

**解决方案**：
- 方案一：在 OmniRouter 代理层添加 middleware 伪造 reasoning_content
- 方案二：换用官方模型（Sonnet/Opus）
