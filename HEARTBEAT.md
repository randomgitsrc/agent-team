# HEARTBEAT.md - 心跳检查规则

## 📋 检查清单（每次心跳执行）

### 1. 任务看板检查
- [ ] 读取 `memory/work/TODO.md`
- [ ] 检查逾期任务（截止<现在且状态≠DONE）
- [ ] 检查阻塞任务（状态=BLOCKED且>1小时）
- [ ] 检查滞留任务（状态未更新>阈值）

### 2. 自动执行（无需请示）
- [x] Git提交 → 执行 `git add . && git commit -m "chore: 心跳提交"`
- [x] 任务状态更新 → 检查TODO.md状态
- [x] 看板刷新 → 清理冗余文件，更新核心文档
- [x] 状态哈希检查 → 检测状态变更
- [x] 质量巡查 → 检查文档是否过时（>7天未更新的核心文档）、脚本是否空壳（只有框架没有实际逻辑）

### 3. 需请示后执行（不自动）
- [ ] 任务新增/取消
- [ ] 优先级变更（P1→P0）
- [ ] 责任人变更
- [ ] 截止日期延期>4h
- [ ] 资源冲突（多P0同时出现）

## 📝 报告格式规则

没有事情的话，不要徒增打扰，直接不发送消息即可。
⚠️ 严禁把"判断过程"当作回复发出去。无事可报 = HEARTBEAT_OK，不发任何消息给用户。

## 📝 发送报告条件
- [x] 检查到有逾期、阻塞、滞留任务
- [x] 有需请示后执行的事项

### 心跳报告必须遵循：
```
## 📊 心跳检查 [时间]

### ✅ 任务看板
- 已完成：X项
  - （如有）列表
- 进行中：Y项 
  - （如有）列表 
- 待办：Z项
  - （如有）列表
- 逾期：M项
  - （如有）列表
- 阻塞：N项
  - （如有）列表

### 🔄 自动执行
- Git提交：已提交N个文件
- 状态更新：已检查
- 看板刷新：已清理/更新
- 状态检测：检测到变更/无变更

### 📈 架构进展
- 关键进展1
- 关键进展2

### 📅 下次检查
[预计时间]
```

### 禁止：
- 文字扎堆（使用空行分隔）
- 过多细节（只报告关键信息）
- 重复内容（相同信息不重复报告）
